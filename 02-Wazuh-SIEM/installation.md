# Wazuh SIEM — Installation

## Prerequisites

- Ubuntu Server 22.04 LTS, minimum 4 vCPU / 8 GB RAM (16 GB recommended for production-scale ingestion)
- Static IP, DNS resolution, NTP time sync configured
- Ports open: 1514/tcp (agents), 1515/tcp (enrollment), 55000/tcp (API), 9200/tcp (indexer), 443/tcp (dashboard)

## All-in-One Install (single node)

```bash
curl -sO https://packages.wazuh.com/4.x/wazuh-install.sh
sudo bash ./wazuh-install.sh -a
```

This installs the Wazuh indexer, manager, and dashboard on the same host and generates admin credentials at the end of the run — save these immediately.

## Verify Services

```bash
sudo systemctl status wazuh-manager
sudo systemctl status wazuh-indexer
sudo systemctl status wazuh-dashboard
```

## Post-Install Hardening

- Rotate the default `admin` password
- Restrict dashboard access (firewall / reverse proxy with TLS)
- Enable Filebeat/Indexer TLS certs (generated automatically by the installer)
- Configure log retention / index lifecycle management (ILM) based on available disk

## Distributed Deployment Note

For larger environments (~300–350 endpoints), a single-node all-in-one setup is not recommended. See `wazuh-configuration.md` for the distributed cluster design (separate indexer cluster, manager cluster behind a load balancer).
